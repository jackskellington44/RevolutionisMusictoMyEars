# Revolution is Music to My Ears

A Getty Images music visualizer — pure automated art. White page, press spacebar, music plays and Getty images swap in two side-by-side panels in sync with the beat.

---

## File Structure

```
RevolutionisMusictoMyEars/
├── index.html          # The entire website (HTML + CSS + JS, self-contained)
├── config.json         # Image IDs + settings (generated by process-links.js)
├── process-links.js    # Node script: reads raw-links.txt, extracts Getty IDs, writes config.json
├── raw-links.txt       # Paste raw Getty embed codes here before running process-links.js
├── audio/              # MP3 files
│   ├── 01 - Gulch - Monkey Gone To Heaven.mp3
│   └── 01 - fakemink - Music and Me.mp3
└── README.md
```

## How to Add Images

1. Go to Getty Images and copy the embed code for any image. It looks like this:

   ```html
   <a id='...' class='gie-single' href='https://www.gettyimages.com/detail/1321139945' ...>Embed from Getty Images</a><script>window.gie=window.gie||function(c){...items:'1321139945'...}</script>
   ```

2. Paste one or more embed codes into `raw-links.txt` (you can paste many at once, even as one big blob).

3. Run the processing script:

   ```bash
   node process-links.js
   ```

   The script extracts the numeric image IDs, deduplicates them, and writes the updated `config.json` while keeping your existing settings and playlist.

---

## How to Add Music

1. Drop MP3 files into the `audio/` folder.
2. Open `config.json` and add the filenames to the `playlist` array:

   ```json
   "playlist": [
     "01 - Gulch - Monkey Gone To Heaven.mp3",
     "my-new-track.mp3"
   ]
   ```

   Tracks play in order and loop back to the beginning when the list ends.

## How to Use the Website

1. Open `index.html` in a browser (locally or on GitHub Pages).
2. The page loads as a blank white screen — nothing happens yet.
3. **Press spacebar** — music starts, two panels appear with Getty images that swap in alternating beats (left → right → left → …).
4. **Press spacebar again** — music pauses, panels freeze exactly as they are.
5. **Press spacebar again** — music and beat-swapping resume.

Spacebar is the only control. There is no visible UI.

## Adjusting Settings

Edit the `settings` block in `config.json`:

```json
"settings": {
  "beatSensitivity": 0.65,
  "flashDuration": 0.3,
  "maxConcurrent": 2
}
```

| Setting | Description |
|---|---|
| `beatSensitivity` | `0.0` – `1.0`. Lower = triggers on quieter sounds. Higher = only loud peaks trigger a swap. |
| `flashDuration` | Currently unused (reserved for future animation effects). |
| `maxConcurrent` | Currently unused (reserved for future multi-panel expansion). |

## Deploying to GitHub Pages

1. Push your repository to GitHub.
2. Go to **Settings → Pages** in your repo.
3. Under **Source**, select **Deploy from a branch** → `main` → `/ (root)`.
4. Save. GitHub Pages will serve `index.html` at `https://<username>.github.io/<repo-name>/`.

`config.json` and the `audio/` folder are fetched relative to the page automatically — no extra configuration needed.
